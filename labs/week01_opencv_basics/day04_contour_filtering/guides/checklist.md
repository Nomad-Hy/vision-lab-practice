# Day 4 Checklist

## 0. 문제 정의

- [ ] 어떤 흰색 영역을 최종 후보로 인정할지 한 문장으로 적었다.
- [ ] 작은 노이즈, 넓은 반사, 이미지 경계를 각각 어떻게 다룰지 정했다.
- [ ] 예상되는 실제 후보 수와 가짜 후보 수를 실험 전에 기록했다.

## 1. 입력과 출력 구조

- [ ] `data/raw/`에 대표 이미지를 넣었다.
- [ ] `outputs/images/`와 `outputs/logs/`의 역할을 이해했다.
- [ ] Day 3 파일을 직접 수정하지 않고 Day 4 입력을 독립적으로 준비했다.

## 2. 공통 모듈

- [ ] 출력 폴더 생성 함수를 구현했다.
- [ ] 입력 이미지 탐색 함수를 구현했다.
- [ ] 이미지 읽기 실패 시 예외를 발생시키도록 구현했다.
- [ ] 이미지 저장 성공 여부를 확인하도록 구현했다.
- [ ] JSON과 CSV 저장 함수의 입력 형태를 확인했다.

## 3. 마스크 준비

- [ ] 컬러 이미지를 grayscale로 변환했다.
- [ ] fixed, Otsu, adaptive 중 사용할 방식을 정했다.
- [ ] morphology 연산과 kernel 크기를 정했다.
- [ ] 관심 영역이 흰색인지 확인했다.
- [ ] 마스크가 단일 채널인지 확인했다.
- [ ] 마스크 값이 이진 형태인지 확인했다.

## 4. Contour 분석

- [ ] contour 입력으로 이진 마스크의 복사본을 사용했다.
- [ ] retrieval mode의 의미를 확인했다.
- [ ] approximation mode의 의미를 확인했다.
- [ ] 전체 contour 수를 기록했다.
- [ ] 각 contour의 면적을 계산했다.
- [ ] 각 contour의 둘레를 계산했다.
- [ ] 바운딩 박스를 계산했다.
- [ ] 중심점을 계산했다.
- [ ] 종횡비를 계산했다.
- [ ] 채움 비율을 계산했다.
- [ ] 이미지 경계 접촉 여부를 계산했다.

## 5. 후보 필터링

- [ ] 최소 면적 규칙을 적용했다.
- [ ] 최대 면적 규칙을 적용했다.
- [ ] 최소 너비와 높이 규칙을 적용했다.
- [ ] 종횡비 범위를 적용했다.
- [ ] 채움 비율 범위를 적용했다.
- [ ] 경계 접촉 후보 처리 방식을 정했다.
- [ ] 한 후보가 여러 규칙에서 탈락할 때 모든 이유를 기록했다.
- [ ] 통과 후보와 탈락 후보를 별도 리스트로 나눴다.

## 6. 시각화

- [ ] 모든 contour 결과를 저장했다.
- [ ] 통과 후보와 탈락 후보를 구분해 표시했다.
- [ ] 후보 번호를 표시했다.
- [ ] 긴 설명 대신 짧은 탈락 코드 또는 범례를 사용했다.
- [ ] 원본·마스크·전체 contour·최종 후보 비교 grid를 만들었다.

## 7. 로그와 비교

- [ ] 후보별 특징을 CSV 또는 JSON으로 저장했다.
- [ ] 필터 단계별 탈락 개수를 기록했다.
- [ ] 약한 필터, 중간 필터, 강한 필터를 비교했다.
- [ ] 처리 시간을 측정했다.
- [ ] 이미지 저장 시간과 순수 처리 시간을 구분했는지 기록했다.

## 8. 실패 분석

- [ ] 작은 실제 후보가 제거된 사례를 기록했다.
- [ ] 반사가 실제 후보로 남은 사례를 기록했다.
- [ ] 하나의 물체가 여러 contour로 갈라진 사례를 확인했다.
- [ ] 여러 물체가 하나로 붙은 사례를 확인했다.
- [ ] 회전 때문에 바운딩 박스가 부정확한 사례를 확인했다.
- [ ] 해상도 변화 시 픽셀 기준이 달라지는 문제를 기록했다.

## 9. 공개 산출물

- [ ] README 대표 이미지를 선택했다.
- [ ] 사용한 규칙과 선택 이유를 표로 정리했다.
- [ ] 실패 사례와 한계를 숨기지 않고 적었다.
- [ ] 면접 질문에 자신의 실험 결과를 근거로 답했다.
