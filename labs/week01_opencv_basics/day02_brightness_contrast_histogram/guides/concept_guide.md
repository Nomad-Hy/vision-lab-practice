# Concept Guide - Brightness, Contrast, Histogram

## 1. 이미지는 숫자 배열이다

사람은 이미지를 그림으로 보지만, 컴퓨터는 숫자 표로 본다. 흑백 이미지에서 픽셀 값은 보통 0부터 255 사이의 숫자다.

```text
0   = 검정
255 = 흰색
```

## 2. 밝기

밝기는 이미지 전체가 얼마나 밝거나 어두운지를 뜻한다.

- 밝기 증가: 픽셀 값이 전체적으로 커진다.
- 밝기 감소: 픽셀 값이 전체적으로 작아진다.

주의할 점은 픽셀 값이 255를 넘으면 정보가 사라질 수 있다는 것이다.

## 3. 대비

대비는 밝은 곳과 어두운 곳의 차이다.

- 대비가 높다: 물체와 배경 차이가 크다.
- 대비가 낮다: 물체와 배경 차이가 작다.

머신비전에서는 대비가 낮으면 물체 분리, 경계 검출, 결함 검출이 어려워진다.

## 4. 히스토그램

히스토그램은 이미지 안에 어떤 밝기 값이 얼마나 많은지 보여주는 그래프다.

```text
왼쪽에 몰림   → 어두운 이미지 가능성
오른쪽에 몰림 → 밝은 이미지 또는 과노출 가능성
좁게 몰림     → 저대비 가능성
넓게 퍼짐     → 밝기 차이가 다양함
```

## 5. 히스토그램 평활화

히스토그램 평활화는 좁게 몰린 밝기 분포를 더 넓게 펴서 대비를 높이는 방법이다.

하지만 항상 좋은 것은 아니다.

- 노이즈가 강조될 수 있다.
- 반사가 더 튀어 보일 수 있다.
- 배경 질감이 결함처럼 보일 수 있다.

## 6. 실무 연결

중소·중견 머신비전/카메라 SW에서는 다음 문제가 자주 발생한다.

- 조명 변화로 이미지가 어두워짐
- 금속 반사가 결함처럼 보임
- 대비 부족으로 경계가 안 잡힘
- 노이즈가 결함처럼 오탐됨
- 과노출로 작은 결함이 사라짐

Day 02의 목표는 이런 문제를 바로 해결하는 것이 아니라, 먼저 이미지 품질을 관찰하고 설명할 수 있게 되는 것이다.
